## üîß Customization Queries

### Adjust Dungeon Quest Rewards
```sql
-- Increase rewards for specific dungeon
UPDATE dc_quest_difficulty_mapping 
SET base_token_reward = base_token_reward + 2,
    base_gold_reward = base_gold_reward * 1.5
WHERE dungeon_id = 9;  -- BRD

-- Make dungeon require group
UPDATE dc_quest_difficulty_mapping 
SET requires_group = 1 
WHERE dungeon_id = 55;  -- Halls of Reflection

-- Increase difficulty multiplier
UPDATE dc_difficulty_config 
SET token_multiplier = 2.50, 
    gold_multiplier = 2.50 
WHERE difficulty_name = 'Mythic';
```

### Activate/Deactivate Quests
```sql
-- Disable a quest
UPDATE dc_quest_difficulty_mapping 
SET is_active = 0 
WHERE quest_id = 700224;  -- Mythic+ quest

-- Enable Mythic+ difficulty
UPDATE dc_difficulty_config 
SET enabled = 1 
WHERE difficulty_name = 'Mythic+';

-- Disable all Mythic quests temporarily
UPDATE dc_quest_difficulty_mapping 
SET is_active = 0 
WHERE difficulty = 'Mythic';
```

### Modify Daily/Weekly Rotation
```sql
-- Add quest to different day
UPDATE dc_daily_quest_token_rewards 
SET token_count = 3 
WHERE quest_id = 700105;  -- Triple rewards for this quest

-- Apply event bonus to all weeklies
UPDATE dc_weekly_quest_token_rewards 
SET bonus_multiplier = 2.0;  -- Double tokens

-- Reset event bonus
UPDATE dc_weekly_quest_token_rewards 
SET bonus_multiplier = 1.0;
```

## üìä Useful Queries

### Check Quest Distribution
```sql
-- Count quests per difficulty
SELECT difficulty, COUNT(*) as quest_count
FROM dc_quest_difficulty_mapping
GROUP BY difficulty;

-- Count quests per expansion
SELECT 
  CASE 
    WHEN dungeon_id BETWEEN 1 AND 19 THEN 'Classic'
    WHEN dungeon_id BETWEEN 20 AND 39 THEN 'TBC'
    WHEN dungeon_id BETWEEN 40 AND 55 THEN 'WotLK'
  END as expansion,
  COUNT(*) as quest_count
FROM dc_quest_difficulty_mapping
GROUP BY expansion;
```

### Check Player Progress
```sql
-- Top 10 players by quest completions
SELECT c.name, COUNT(dqc.quest_id) as total_quests
FROM dc_character_dungeon_quests_completed dqc
JOIN characters c ON c.guid = dqc.guid
GROUP BY dqc.guid
ORDER BY total_quests DESC
LIMIT 10;

-- Player difficulty distribution
SELECT 
  difficulty,
  SUM(total_completions) as completions
FROM dc_character_difficulty_completions
WHERE guid = ?  -- Player GUID
GROUP BY difficulty;
```

### Check System Health
```sql
-- Find quests without difficulty mapping
SELECT dqtr.quest_id
FROM dc_daily_quest_token_rewards dqtr
LEFT JOIN dc_quest_difficulty_mapping qdm ON dqtr.quest_id = qdm.quest_id
WHERE qdm.quest_id IS NULL;

-- Find inactive quests
SELECT quest_id, difficulty, dungeon_id
FROM dc_quest_difficulty_mapping
WHERE is_active = 0;

-- Check token distribution
SELECT token_item_id, COUNT(*) as quest_count
FROM dc_daily_quest_token_rewards
GROUP BY token_item_id;
```

## üìù Notes for Adjustments

### When Adding New Dungeons
1. Insert into `dc_dungeon_quest_mapping`
2. Create quest IDs (use next available in range)
3. Add to `dc_daily_quest_token_rewards` or `dc_weekly_quest_token_rewards`
4. Add to `dc_quest_difficulty_mapping`
5. Update this reference document!

### When Creating New Difficulties
1. Add to `dc_difficulty_config`
2. Update difficulty ENUM in table definitions
3. Create new quest ID range
4. Add difficulty-specific achievements
5. Update C++ code to handle new difficulty

### When Adding New Achievements
1. Use next available ID in 10800-10999 range
2. Add to `achievement` table
3. Create title if needed in `chartitles`
4. Update C++ achievement checking code
5. Document in this reference
